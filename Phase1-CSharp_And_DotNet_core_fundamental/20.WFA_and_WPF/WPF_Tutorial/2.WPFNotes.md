# XAML(eXtensible Application Markup Language)
XAML, which stands for eXtensible Application Markup Language, is Microsoft's variant of XML for describing a 
GUI.

XAML is an XML-based declarative language. A WPF window file usually looks like MainWindow.xaml with a matching 
MainWindow.xaml.cs.

Standard root for a window:

<Window x:Class="MyApp.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        Title="My App" Height="450" Width="800">
  <!-- UI here -->
</Window>

# Layout system ‚Äî quick overview
--------------------------------
WPF layout is declarative and adaptive. Instead of setting explicit pixel positions (like WinForms), you 
compose UI with panels that measure and arrange their children during two layout passes: Measure (how big 
children want to be) and Arrange (where children are placed). Understanding the panels and their sizing 
behavior is the key to building robust, DPI-friendly, resizable UIs.

## Grid (the workhorse)

What it is: a table of rows and columns ‚Äî the most commonly used panel for forms and complex layouts.
This is the default choice for professional WPF layouts.

Sizing modes:
- Auto ‚Äî size to content.
- Pixel (100) ‚Äî fixed pixels.
- Star (*, 2*) ‚Äî proportional sizing of remaining space.

Commonly Used Properties :
1. Grid properties

| Property            | Explanation                     | Use case            |
| ------------------- | ------------------------------- | ------------------- |
| `RowDefinitions`    | Defines rows                    | Forms, dashboards   |
| `ColumnDefinitions` | Defines columns                 | Label-value layouts |
| `ShowGridLines`     | Shows grid borders (debug only) | Layout debugging    |

2. RowDefinition / ColumnDefinition

| Property                | Explanation                    |
| ----------------------- | ------------------------------ |
| `Height` / `Width`      | `Auto`, `*`, `2*`, fixed value |
| `MinHeight`, `MaxWidth` | Prevent too small/large rows   |

3. Attached properties (on child controls)

| Property          | Explanation           |
| ----------------- | --------------------- |
| `Grid.Row`        | Row index             |
| `Grid.Column`     | Column index          |
| `Grid.RowSpan`    | Span multiple rows    |
| `Grid.ColumnSpan` | Span multiple columns |

Example Form layout 

<Grid Margin="10" ShowGridLines="False">
    <Grid.ColumnDefinitions>
        <ColumnDefinition Width="150"/>
        <ColumnDefinition Width="*"/>
    </Grid.ColumnDefinitions>

    <Grid.RowDefinitions>
        <RowDefinition Height="Auto"/>
        <RowDefinition Height="Auto"/>
        <RowDefinition Height="*"/>
    </Grid.RowDefinitions>

    <TextBlock Text="Name:" Grid.Row="0" Grid.Column="0" VerticalAlignment="Center"/>
    <TextBox Grid.Row="0" Grid.Column="1" Margin="4"/>

    <TextBlock Text="Email:" Grid.Row="1" Grid.Column="0" VerticalAlignment="Center"/>
    <TextBox Grid.Row="1" Grid.Column="1" Margin="4"/>

    <Button Content="Save"
            Grid.Row="2"
            Grid.Column="1"
            HorizontalAlignment="Right"
            Width="100"
            Margin="4"/>
</Grid>

explain :  

            Name : <Textboc>
            email : <Textbox>

                <SaveButto>

Use cases

- Forms
- CRUD screens
- Dashboards
- Any resizable UI

üìå Rule: If you‚Äôre confused ‚Üí use Grid.

Pitfalls / tips:

- Don‚Äôt create a deeply nested grid for every row ‚Äî moderate nesting is fine, but flatten when possible.
- Use Star sizing for responsive resizing; use Auto where controls dictate size.
- Use SharedSizeGroup when you want columns across separate Grids to line up (e.g., labels aligned in stacked 
forms).

Example : SharedSizeGroup and SharedSizeScope 

You can force two columns, or rows, in a WPF Grid Panel to be the same size. How do you do that? Use the 
SharedSizeScope and SharedSizeGroup properties. When defining the columns (or rows) use SharedSizeGroup and use 
SharedSizeScope when defining the Grid.

## StackPanel

What it is: stacks children horizontally or vertically in order. Arranges controls in a straight line (vertical 
or horizontal).

Commonly Used Properties : 
| Property              | Explanation               | Use case        |
| --------------------- | ------------------------- | --------------- |
| `Orientation`         | `Vertical` / `Horizontal` | Toolbars, forms |
| `HorizontalAlignment` | Aligns whole panel        | Center group    |
| `VerticalAlignment`   | Aligns panel vertically   | Footer buttons  |

Note : Child controls ignore available space in stack direction.

Example : 

<StackPanel Orientation="Vertical" Margin="10">
    <TextBlock Text="Login" FontSize="18"/>
    <TextBox Margin="0,5"/>
    <PasswordBox Margin="0,5"/>
    <Button Content="Sign In" Width="120"/>
</StackPanel>

explain : 

        Login  
        <Textbox> 
        <passwordbox>
        <Button Sign in>

Use cases :

- Login forms
- Vertical menus
- Button groups

üö´ Avoid for large resizable forms.

Pitfalls / tips:

- StackPanel gives children infinite space in the stacking direction during measure; not ideal for 
  virtualization or huge item lists (use VirtualizingStackPanel inside ItemsControl for that).
- Use Margin on children for spacing; StackPanel has no Spacing property in older WPF versions.

## DockPanel
What it is: docks children to the top/left/right/bottom. By default the last child fills the remaining space 
(LastChildFill=true).

Commonly Used Properties : 

1. DockPanel properties
| Property        | Explanation                                         |
| --------------- | --------------------------------------------------- |
| `LastChildFill` | Last control fills remaining space (default = true) |

2. Attached property
| Property         | Explanation                      |
| ---------------- | -------------------------------- |
| `DockPanel.Dock` | `Top`, `Bottom`, `Left`, `Right` |

Example (Application Shell) : 

<DockPanel>

    <Menu DockPanel.Dock="Top"/>
    <ToolBar DockPanel.Dock="Top"/>

    <StatusBar DockPanel.Dock="Bottom"/>

    <StackPanel DockPanel.Dock="Left" Width="200">
        <Button Content="Dashboard"/>
        <Button Content="Reports"/>
    </StackPanel>

    <Grid>
        <!-- Main Content -->
    </Grid>

</DockPanel>

Use cases

- Main window layout
- Navigation + content
- Menu / status bar apps

## WrapPanel
What it is : Places controls left to right, then wraps to next row when space ends.

places children in a line and wraps them into new rows/columns when space runs out.

Commonly Used Properties : 
| Property      | Explanation           |
| ------------- | --------------------- |
| `Orientation` | Horizontal / Vertical |
| `ItemWidth`   | Fixed width for items |
| `ItemHeight`  | Fixed height          |

example :

    <Button Content="One" Width="100" Margin="5"/>
    <Button Content="Two" Width="100" Margin="5"/>
    <Button Content="Three" Width="100" Margin="5"/>

- Thumbnail galleries, tag lists, wrapping toolbar

Pitfalls / tips:

- StackPanel to arrange items
